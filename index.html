CSS
寬度繼承

若一個Block沒有設定寬度，則會自動與父項目同寬。

間隔優先權

無論如何設定Padding與Margin，一個子項目永遠無法影響父項目的已設定寬高。

選擇器與空格

若選擇器間有空格，則表示為選擇最後一項，選擇器間有父子關係。

.example .result{}

若選擇器間無空格，則表示套用此Class的元素需同時包含所有選擇器，並且此元素還能有其他Class存在。
常用於jQuery的Class相關指令。

.example.result{}
選擇器與Hover

能特別選擇在Hover狀態下的子項目。
常用於單CSS的動畫效果。

.example:hover .result{}
選擇器與順序關係

若選擇器間有>，即代表選擇往下一層的所有該子樣式。

.example .result > li{}

選擇器間的>能夠堆疊指定。

.example .result > li > li{}
Padding與Margin

盡量在外層設定統一數值，方便管理。

Indent與層

當一個外層設有Indent時，內層的Indent若低於外層數值則不會顯示任何變化。

Float

需接著設置Clearfix。

使用A設定各區塊之樣式，並透過自適應設定總寬高時，須注意Padding、Display與Height的互相關係、注意有無偏離。

1↑↓Padding與Height二擇一。
2兩者共用時，在使用Height與Line-height來使文字置中時需先計算、使Line-height的數值為Height減去↑↓Padding總和。

鏈式寫法

一段程式能有多個動作，jQuery會依照順序、完成一項後接著執行下一項，直到最後。注意Show與Hide類需有持續時間。

RWD
響應式轉換注意事項
1文字溢出時需注意Word-wrap與Word-break。
2左右的推擠可使用%值，但上下的推擠因載具高度不一致、需使用Px為單位。
3上下推擠需遵循Vertical Rhythm。
4若有需滿版的寬度，不可直接盲用100%，須使Width、Margin與Padding的左右總值合為100%。
5響應式轉換不可能使每個解析度最優化，選擇最常見的寬度進行響應式處理即可。
6選擇響應式的寬度斷點時，若有舊網站，應先分析舊網站資料來判斷斷點，若無則使用最熱門寬度即可。
7Desktop Frist，先由電腦版做起，之後使用Max-Width漸漸向下相容不同寬度。特點是電腦版會先固定寬度，到行動版單欄時才使用%。
8Mobile First，先由行動版做起，之後使用Min-Width漸漸向上相容不同寬度。特點是先用%製作，到一定寬度以後的版本才使用固定寬度。
9最小寬度至少要有320px，即iPhone5與SE的寬度。
Vertical Rhythm

又稱垂直韻律。使圖片、段落、間隔推擠的比例能適宜閱讀。

1根據字體大小設定行距。
2行距需能使字體置中。
3圖片、段落的長寬須能被行距整除。
Margin-top問題

當Margin-top上下區塊重疊時，有以下方法可解。

1以上一層外元素的Padding代替推擠。
2上一層外元素加入Border。
3出現問題的元素的Display改為Incline-blcok。
4統一使用Margin-bottom。
RWD細節

以UI與UX設計角度切入所該注重的內容。

1注意動線，並非所有內容都需另製至響應式版面，某些次要內容，如廣告類可以捨棄，或者排於主要內容後。
2因肉眼可見文字有限，應以一行的文字數量決定何時該使用何種版型，如三欄、單欄等等。
3注意點選範圍，過窄的點選範圍會降低使用者體驗，通常而言，人類食指的點選寬度至少要有44px。
4響應式版面上的內容應精簡，如若大量資訊同時有標題、描述、內文，轉至響應式時可以將內文隱藏。
5一般版面可使用Hover反應，但響應式宜用Touch，以免使用者被自身肢體擋住視線、或者因Hover顯示時間不夠長而感繁瑣。
6行動版的主選單點擊顯示方式應只配合行動版，而一般版則盡可能利用空間。在當中，重複的元素透過Display與CSS、JS來控制顯示與否或顯示方式。
7需注意字型寬度。如在320px寬度中理論上可以有20個16px的字，但因間隙，實際上只能容納12個左右的字，設計一開始就需要考慮每個選單的字數。
8撰寫Media Query時，在一個段落後寫該斷落的斷點，不要每行CSS夾雜斷點或全部斷點後才統一寫Media Query。

2單獨在表格的父階層套用Over-Flow:Auto，使表格可以被單獨左右滑動而不影響其他版面，並且單獨改變表格的固定寬度，使文字不會被斷行。

Mixin

一套CSS的組合，可用來記錄常用技術，不必每次使用時回想。

1使用@mixin 名字 {CSS內容}來宣告。CSS內容可有多重{}，尤其在@media或@content的應用時。
2可配合Variable，如@mixin 名字($變量1,$變量2){CSS內容，當中數值應用變量}。
3CSS內容中加入@content;，能使在載入mixin後，在括號內再加入CSS內容。如@include mixin檔名{新的CSS內容};。通常用於有&的mixin或對螢幕寬度的@media等。
4配合&，&會自動變更為當下呼叫該Mixin的CSS選擇器，覆蓋較高權限、統一選擇器的用法為@mixin 名字{.選擇器 & {@content}}。
5配合&，&會自動變更為當下呼叫該Mixin的CSS選擇器，狀態選擇器的用法為@mixin 名字{&:link,&:visited {@content}}。
6使用@include mixin檔名;載入。

Media Query
Max-Width

區塊內程式碼會在指定最大寬度以下或相等的螢幕寬度生效。因應CSS同層級越下越優先的特性，應由螢幕寬至窄的順序由上往下編寫。最上則為電腦用。

body{padding:30px;}
@media(max-width:1024px){body{padding:20px;}}
@media(max-width:768px){body{padding:10px;}}
@media(max-width:375px){body{padding:5px;}}
Min-Width

區塊內程式碼會在指定最小寬度以上或相等的螢幕寬度生效。因應CSS同層級越下越優先的特性，應由螢幕窄至寬的順序由上往下編寫。最上則為電腦用。

body{padding:30px;}
@media(min-width:375px){body{padding:5px;}}
@media(min-width:768px){body{padding:10px;}}
@media(min-width:1024px){body{padding:20px;}}
Max-Width and Min-Width

利用Max-Width指定以下寬度、再利用Min-Width指定以上寬度，藉此指定某個特定區間，常用於已經設定主流寬度後、某個特別需要調整的寬度區段。

body{padding:30px;}
@media(max-width:1024px) and (min-width:569px;){body{padding:20px;}}
CSS
Max-Width

限制寬度最大值，但當縮小時有自適應效果，寬度比最大值小時會變更寬度、自動將內容排下，而非如傳統Width般固定寬度。

body{max-width:800px;}
%單位

以父元素的寬度為標準取值。若非固定寬度，父元素的寬度宜用Max-Width。

body{max-width:800px;}.menu{width:30%;}.content{width:70%;}

左側固定、右側流體式設計

利用固定寬度與Float設定左側欄，再於右側欄設定大於該值的Margin-Left，以取父元素寬度為原理單使右側欄作響應式。最後須於所有欄後添加ClearFix。

.left-column {width:160px;float:left;border:1px solid blue;padding:1%;margin:1%;margin-top:16px;margin-bottom:16px;}
.right-column {margin-top:16px;margin-bottom:16px;margin-left:197.8px;border:1px solid blue;padding:1%;}

窄於一定寬度後，取消Float與Margin，變為單欄式，從而避免文字過擠的情形。

@media(max-width:1080px){.left-column {float:none;width:96%;margin:16px auto;}}
@media(max-width:1080px){.right-column {margin-left:1%;}}
